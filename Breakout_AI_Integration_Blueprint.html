0. Why Groq — Free Tier Limits at a Glance

Groq offers a free tier ("Developer" plan) that covers all the AI needs of the Fireteam platform at typical session volumes. Both Whisper and LLM inference run on the same API key.

Audio Capture Strategy (Browser Side)

How audio gets from LiveKit to Whisper: LiveKit exposes each participant's audio track via the JS SDK. Attach a MediaRecorder to each remote participant's audio track, collect 30-second WebM blobs, POST to your Laravel backend. Your backend queues a TranscribeAudioJob that sends the blob to Groq Whisper and appends the result to fireteam.transcripts. 
// React (browser) — attach recorder to LiveKit participant audio
const participant = room.getParticipantByIdentity(userId);
const audioTrack = participant.audioTrackPublications.values().next().value?.track;
const stream = new MediaStream([audioTrack.mediaStreamTrack]);
const recorder = new MediaRecorder(stream, { mimeType: 'audio/webm' });

recorder.ondataavailable = async (e) => {
  if (e.data.size > 0) {
    const formData = new FormData();
    formData.append('audio', e.data, 'chunk.webm');
    formData.append('session_id', sessionId);
    formData.append('slide_id', currentSlideId);
    formData.append('user_id', participantUserId);
    await fetch('/api/transcribe', { method: 'POST', body: formData });
  }
};
recorder.start(30000); // 30-second chunks

3. Database Changes — New Tables Needed

Add these tables to the fireteam PostgreSQL schema. The migration file is included in the code deliverables.

-- Raw audio transcripts (per user, per slide, per chunk)
CREATE TABLE fireteam.transcripts (
    id              BIGSERIAL PRIMARY KEY,
    session_id      BIGINT NOT NULL REFERENCES fireteam.sessions(id) ON DELETE CASCADE,
    experience_id   BIGINT NOT NULL REFERENCES fireteam.experiences(id),
    slide_id        BIGINT NOT NULL REFERENCES fireteam.slides(id),
    user_id         UUID NOT NULL,
    chunk_index     INT NOT NULL DEFAULT 0,       -- order within slide
    transcript_text TEXT NOT NULL,
    duration_seconds NUMERIC(8,2),
    whisper_model   VARCHAR(50) DEFAULT 'whisper-large-v3',
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(session_id, slide_id, user_id, chunk_index)
);

-- AI evaluation results (Bloom's Taxonomy scores per student per rubric)
CREATE TABLE fireteam.rubric_results (
    id              BIGSERIAL PRIMARY KEY,
    session_id      BIGINT NOT NULL REFERENCES fireteam.sessions(id) ON DELETE CASCADE,
    rubric_id       BIGINT NOT NULL REFERENCES fireteam.rubrics(id),
    user_id         UUID NOT NULL,
    score           SMALLINT NOT NULL CHECK (score BETWEEN 0 AND 6),
    arguments       JSONB NOT NULL DEFAULT '[]',  -- string[] evidence quotes
    justification   TEXT NOT NULL DEFAULT '',
    disagreements   JSONB NOT NULL DEFAULT '[]',
    groq_model      VARCHAR(100),
    prompt_tokens   INT,
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(session_id, rubric_id, user_id)
);

-- Post-session markdown summaries
CREATE TABLE fireteam.session_summaries (
    id              BIGSERIAL PRIMARY KEY,
    session_id      BIGINT NOT NULL UNIQUE REFERENCES fireteam.sessions(id) ON DELETE CASCADE,
    summary_md      TEXT NOT NULL,
    group_balance_score NUMERIC(4,2),            -- 0.0 - 1.0
    groq_model      VARCHAR(100),
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Insight tags / themes extracted from group discussion
CREATE TABLE fireteam.session_insights (
    id              BIGSERIAL PRIMARY KEY,
    session_id      BIGINT NOT NULL REFERENCES fireteam.sessions(id) ON DELETE CASCADE,
    insight_type    VARCHAR(50) NOT NULL,         -- 'theme', 'key_concept', 'question_raised'
    label           TEXT NOT NULL,
    relevance_score NUMERIC(4,3),                -- 0.0 - 1.0
    created_at      TIMESTAMPTZ DEFAULT NOW()
);

-- Index for fast transcript assembly
CREATE INDEX idx_transcripts_session_user ON fireteam.transcripts(session_id, user_id);
CREATE INDEX idx_transcripts_slide ON fireteam.transcripts(slide_id, user_id);
CREATE INDEX idx_rubric_results_session ON fireteam.rubric_results(session_id);

5. Feature A: Live Session Transcription (Whisper)

AI Service: Groq Whisper API — POST /audio/transcriptions using model whisper-large-v3

How It Works

  1. During discussion slides (slideType 6), the browser starts a MediaRecorder for each participant's audio track from LiveKit.
  2. Every 30 seconds, the browser POSTs the audio chunk to /api/transcribe with the user ID, session ID, and slide ID.
  3. Laravel queues a TranscribeAudioJob — the job calls GroqWhisperService::transcribe().
  4. Groq returns the text. The job appends it to fireteam.transcripts with the chunk index.
  5. When the session ends, EvaluateSessionJob fetches all chunks for each user ordered by chunk_index and concatenates them into a full transcript.

Only Transcribe Discussion Slides

Only run MediaRecorder during slideType 6 (discussion prompt) slides. Stop it on slide advance. This keeps Groq audio quota usage lean — you capture only the content that matters for evaluation. 
// Frontend: watch activeSlide from room_state
useEffect(() => {
  if (currentSlide?.slideType === 6) {
    startRecording(participants); // attach MediaRecorder to each participant
  } else {
    stopAllRecorders();           // stop + flush last chunk
  }
}, [activeSlide]);

Laravel API Endpoint

// routes/api.php
Route::post('/transcribe', [TranscriptionController::class, 'store'])
    ->middleware('auth:sanctum');

// TranscriptionController::store
public function store(Request $request): JsonResponse
{
    $request->validate([
        'audio'      => 'required|file|mimes:webm,mp4,wav,ogg|max:25600', // 25MB Groq limit
        'session_id' => 'required|integer|exists:fireteam.sessions,id',
        'slide_id'   => 'required|integer|exists:fireteam.slides,id',
        'user_id'    => 'required|uuid',
    ]);

    $path = $request->file('audio')->store('tmp/audio');

    TranscribeAudioJob::dispatch(
        storagePath: $path,
        sessionId:   $request->session_id,
        slideId:     $request->slide_id,
        userId:      $request->user_id,
    );

    return response()->json(['queued' => true]);
}

6. Feature B: Bloom's Taxonomy Evaluation (Groq LLM)

AI Service: Groq LLM — POST /chat/completions using model llama-3.3-70b-versatile
Trigger: Session reaches slideType 11 (processingData) → EvaluateSessionJob dispatched

Prompt Architecture (Proven from prompt_type.json)

The production Breakout platform uses four prompt sections combined into one system message:

SYSTEM PROMPT = primaryPrompt + outputPrompt + statsPrompt + reminderPrompt

primaryPrompt:  Role definition + Bloom's Taxonomy explanation + task description
outputPrompt:   JSON schema the model MUST return
statsPrompt:    Instruction to compute score distribution
reminderPrompt: "Be strict. Score 0 if not discussed. Use only evidence from transcript."

USER MESSAGE = """

{concatenatedTranscriptText}

7. Feature C: Session Summary Generation (Groq LLM)

AI Service: Groq LLM — model llama3-8b-8192 (faster, cheaper for this simpler task)
Trigger: After EvaluateSessionJob completes

What It Produces

A markdown document stored in fireteam.session_summaries that includes: what the group discussed, how well they covered the key concepts, group balance score (who dominated vs. who was quiet), and top 3 insights.

Prompt

SYSTEM: You are an educational analytics assistant. Write a concise session summary
for an instructor reviewing their students' discussion performance.

USER:
Case Study: {slideDeckName}
Session Date: {scheduledAt}
Participants: {participantNames}

Full Group Transcripts (by participant):
{allTranscriptsFormatted}

Rubric Scores:
{rubricResultsSummary}

Write a markdown summary with these sections:
## Overview
## What the Group Discussed Well
## Gaps in Coverage
## Participation Balance
## Top 3 Insights

Also return a group_balance_score between 0.0 (one person dominated) and 1.0 (perfectly even).

8. Feature D: Quiz Explanation Generation (Groq LLM)

AI Service: Groq LLM — model llama3-8b-8192
Trigger: Instructor saves a new question in the admin panel (or clicks "Generate Explanation")

What It Does

The explanation field on each quiz question (shown to students when they pick the wrong answer) is expensive to write manually. Groq generates it automatically from the question + case study context.

// Laravel: QuestionObserver or explicit controller action
public function generateExplanation(Question $question): string
{
    $experience = $question->slide->experience;

    $prompt = "You are a learning coach for a business case study titled '{$experience->name}'.
The pre-work quiz question is: '{$question->question_text}'
The correct answer is: '{$question->answers[$question->correct_answer_index]}'
The incorrect options are: " . implode(', ', $question->wrongAnswers()) . "

Write 2–3 sentences explaining WHY the correct answer is right,
in plain language a student can understand after getting it wrong.
Be encouraging and educational, not just factual.";

    return $this->groq->chat([
        'model'    => 'llama3-8b-8192',
        'messages' => [['role' => 'user', 'content' => $prompt]],
    ]);
}
UX tip: Add a "✨ Generate Explanation" button next to the explanation textarea in the instructor question editor. This calls POST /api/admin/questions/{id}/generate-explanation and fills the field. The instructor can edit before saving.

9. Feature E: Video Caption Seeding (Whisper)

AI Service: Groq Whisper API with timestamp_granularities: ["segment"]
Trigger: Instructor uploads a video to a slide (admin panel)

What It Does

When an instructor uploads a video for a slide, Whisper generates timed caption segments matching the fireteam.slide_captions schema (caption, start_time, end_time). This populates accessibility captions and provides the phrase set for AI evaluation.

Key: Use Verbose JSON Response

// GroqWhisperService::transcribeWithTimestamps()
$response = Http::withToken($this->apiKey)
    ->attach('file', file_get_contents($videoPath), basename($videoPath))
    ->post($this->baseUrl . '/audio/transcriptions', [
        'model'                    => 'whisper-large-v3',
        'response_format'          => 'verbose_json',   // ← get timestamps
        'timestamp_granularities'  => 'segment',        // sentence-level segments
        'language'                 => 'en',
    ]);

// Response shape:
// { "segments": [{ "text": "...", "start": 6.0, "end": 9.0 }, ...] }

// Map to slide_captions:
foreach ($response['segments'] as $seg) {
    SlideCaption::create([
        'slide_id'       => $slide->id,
        'experience_id'  => $slide->experience_id,
        'caption'        => trim($seg['text']),
        'start_time'     => $seg['start'],
        'end_time'       => $seg['end'],
    ]);
}
File size limit: Groq Whisper accepts up to 25MB per request. For longer videos, split into chunks using ffmpeg before sending. A 25MB WebM/MP4 at typical quality covers ~15–20 minutes of video.

Retry + Fallback Strategy in Laravel

// In GroqBaseService.php
protected function callWithRetry(array $payload, string $endpoint): array
{
    $attempts = 0;
    $maxAttempts = 3;

    while ($attempts < $maxAttempts) {
        $response = Http::withToken($this->apiKey)
            ->timeout(60)
            ->post($this->baseUrl . $endpoint, $payload);

        if ($response->status() === 429) {
            // Rate limited — back off exponentially
            $retryAfter = $response->header('Retry-After', 60);
            sleep((int) $retryAfter);
            $attempts++;
            continue;
        }

        if ($response->failed()) {
            // Try fallback model
            if ($payload['model'] === 'llama-3.3-70b-versatile') {
                $payload['model'] = 'mixtral-8x7b-32768';
                $attempts++;
                continue;
            }
            throw new GroqApiException($response->body());
        }

        return $response->json();
    }

    throw new GroqApiException('Max retry attempts exceeded');
}
